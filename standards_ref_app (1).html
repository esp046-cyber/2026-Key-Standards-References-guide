IEC 61850
IEC International
‚ñº
Communication networks and systems for power utility automation
üìã Overview
Purpose
Defines communication protocols and data models for substation automation systems (SAS), enabling interoperability between intelligent electronic devices (IEDs) from different manufacturers.
Global Significance
The worldwide standard for substation automation, replacing proprietary protocols with open, standardized communication.
üîß Key Components
Part 7-1: Principles & Models
Establishes the abstract communication service interface (ACSI) and common data classes (CDC) that form the foundation of the standard.
Part 8-1: MMS Mapping
Manufacturing Message Specification (MMS) protocol mapping for real-time substation communication over TCP/IP networks.
Part 9-2: GOOSE
Generic Object Oriented Substation Event - fast peer-to-peer messaging for protection and control (4ms response time).
Part 9-3: Sampled Values
Digital transmission of voltage and current measurements, enabling process bus architecture and non-conventional instrument transformers.
üí° Key Features
  • Self-description: IEDs can describe their capabilities and data structures automatically
  • Configurability: SCL (Substation Configuration Language) XML-based engineering
  • High-speed communication: Type 1A GOOSE messages within 3-4 milliseconds
  • Process bus: Merging units digitize analog signals at source
  • Time synchronization: IEEE 1588 PTP for microsecond accuracy
üéØ Applications
  • Substation automation and protection systems
  • SCADA integration and control centers
  • Distributed energy resources (DER) management
  • Renewable energy integration (wind farms, solar)
  • Electric vehicle charging infrastructure
  • Microgrid and smart grid implementations
Latest Edition
IEC 61850 Edition 2.1 (2020) includes enhancements for data modeling, cybersecurity (IEC 62351 integration), and expanded use cases beyond substations.
Current version supports both station bus and process bus architectures
IEC 60870-5-104
IEC SCADA
‚ñº
Telecontrol equipment and systems - Network access for IEC 60870-5-101
üìã Overview
Purpose
Extends IEC 60870-5-101 serial communication protocol to TCP/IP networks, enabling remote control and monitoring of substations and RTUs over standard IP infrastructure.
Protocol Type
Master-slave architecture telecontrol protocol designed for SCADA systems, widely deployed in European and Asian power systems.
üîß Technical Specifications
Transport Layer
TCP/IP Port 2404 (standard), providing reliable transmission with connection-oriented communication.
Data Types Supported
  • Single point information (SPI) - binary states
  • Double point information (DPI) - 4-state values
  • Measured values - normalized, scaled, short floating point
  • Integrated totals - energy counters
  • Commands - single/double point, setpoint, regulating
  • File transfer - disturbance records, sequences of events
Information Object Addressing
Common Address (CA): 1-65535 identifying stations; Information Object Address (IOA): 1-16777215 identifying data points.
üí° Key Features
  • Spontaneous transmission: Event-driven reporting reduces network traffic
  • Clock synchronization: Built-in time stamping for sequence of events (SOE)
  • Cyclic data transmission: Periodic integrity scans for data validation
  • Command execution: Select-before-execute mechanism with acknowledgment
  • Quality descriptors: 8-bit quality flags for data validity assessment
  • Test mode: Built-in test frames for system diagnostics
üéØ Applications
  • Distribution management systems (DMS)
  • Transmission SCADA control centers
  • RTU and substation gateway communication
  • Renewable energy plant monitoring
  • Industrial automation and process control
  • Water and wastewater SCADA systems
‚ö° Performance Characteristics
Typical Response Times
Status changes: 50-200ms; Command execution: 100-500ms; Cyclic data: 1-60 seconds configurable intervals.
Scalability
Supports thousands of data points per RTU; multiple parallel connections; suitable for both small substations and large control centers.
Cybersecurity Considerations
While IEC 104 lacks native encryption, it's commonly secured using VPNs, TLS/SSL tunnels, or IEC 62351 security extensions. Firewall rules and network segmentation are essential.
Edition 2.0 (2006) remains current; companion standard IEC 60870-5-101 covers serial communication
AS/NZS 3000:2018
Australian/NZ
‚ñº
Electrical installations (known as the Australian/New Zealand Wiring Rules)
üìã Overview
Legal Status
Mandatory standard referenced in state/territory electrical safety legislation. Compliance is legally required for all electrical work in Australia and New Zealand.
Scope of Coverage
Governs design, construction, verification and operation of electrical installations up to 1000V AC (1500V DC), including consumer mains, submains, final subcircuits, and associated equipment.
üîß Major Sections
Section 1: Scope and Application
Defines what installations are covered, exemptions (mining, railways, ships), and relationship with other standards and regulations.
Section 2: Protection & Safety
  • Protection against electric shock (basic & fault protection)
  • RCD requirements: 30mA for socket outlets, portable equipment
  • Protective earthing and equipotential bonding
  • Arc fault detection devices (AFDD) for specific applications
  • Surge protection device (SPD) requirements
Section 3: Conductors & Cables
  • Current carrying capacity tables and derating factors
  • Cable sizing for voltage drop (‚â§5% for submains, ‚â§3% lighting)
  • Minimum conductor sizes (1.5mm¬≤ for power, 1.0mm¬≤ lighting)
  • Color coding: Brown/Black (active), Blue (neutral), Green/Yellow (earth)
Section 4: Switchgear & Protection
Main switches, distribution boards, circuit breakers, fuses, isolation requirements, labeling and identification.
Section 5: Installation Methods
Wiring systems, cable routes, conduit/trunking, support spacing, proximity to other services, underground installations.
Section 6: Special Installations
  • Bathrooms and wet areas (zones 0, 1, 2)
  • Swimming pools and spas
  • Medical locations (not operating theaters)
  • Marinas and boats
  • Caravans, camping sites, similar locations
Section 7: Equipment
Socket outlets, lighting, heating, motors, transformers, EV charging, battery storage systems, PV solar installations.
Section 8: Verification
Inspection and testing requirements, insulation resistance (‚â•1MŒ©), earth continuity, polarity, RCD testing (trip time ‚â§300ms at 30mA).
üí° Key Requirements
Earthing Systems
  • TN-C-S (MEN system) - most common in urban areas
  • TN-S - separate neutral and earth from transformer
  • TT - local earth electrode, rural installations
  • Main earthing conductor ‚â•16mm¬≤ minimum
Circuit Protection
Maximum Zs (earth fault loop impedance) values ensure disconnection within 0.4s (final circuits) or 5s (distribution). Protective device rating must coordinate with cable capacity.
üéØ Special Applications
Solar PV Systems
Requirements for DC isolators, AC/DC labeling, array installation, inverter placement, overvoltage protection, and firefighter safety (rapid shutdown).
EV Charging
Dedicated circuits, socket outlet types (15A, 32A), hardwired EVSE requirements, load management, earthing for outdoor installations.
Battery Energy Storage
Ventilation requirements, temperature monitoring, short circuit protection, isolation from grid, emergency shutdown provisions.
2018 Amendment Highlights
Introduction of AFDD requirements for certain installations, updated solar PV rules, enhanced EV charging provisions, revised bathroom zone classifications, improved surge protection guidance.
Current edition: AS/NZS 3000:2018 (Amendment 2 - 2023); Next revision expected 2025-2026

AS 2067:2016
Australian HV
‚ñº
Substations and high voltage installations exceeding 1 kV a.c.
üìã Overview
Scope
Covers design, construction, operation and maintenance of AC substations and high voltage installations exceeding 1 kV up to and including 765 kV. Applicable to transmission, distribution, industrial and generation substations.
Voltage Classifications
  • Low Voltage (LV): Up to 1 kV
  • High Voltage (HV): >1 kV to 66 kV
  • Sub-Transmission: 66 kV to 132 kV
  • Transmission: 132 kV, 220 kV, 275 kV, 330 kV, 500 kV
üîß Design Requirements
Electrical Clearances
Minimum phase-to-phase, phase-to-earth, and working clearances for various voltage levels. Example: 11 kV requires 150mm phase-to-earth, 22 kV requires 230mm, 66 kV requires 600mm.
Short Circuit Ratings
Equipment must withstand thermal and mechanical stresses during fault conditions. Fault levels calculated per AS 60909 series. Typical distribution substation: 250-500 MVA, transmission: 3000-8000 MVA.
Earthing System Design
  • Earth grid/mat to limit touch and step potentials to safe levels
  • Earth potential rise (EPR) calculations required for fault conditions
  • Touch voltage limit: typically 75V + (1000 √ó ts √ó Cs) where ts = fault clearing time
  • Step voltage considerations for outdoor yards
  • Earth resistance targets: <1Œ© transmission, <5Œ© distribution substations
Insulation Coordination
Lightning impulse withstand, switching impulse, power frequency withstand voltages. Surge arresters rated per AS 60099 to protect equipment from overvoltages.
üí° Physical Layout
Substation Configurations
  • Single busbar - simple, lower cost, limited flexibility
  • Main and transfer bus - operational flexibility
  • Double busbar - high reliability, maintenance capability
  • Ring bus - N-1 security, no main bus
  • Breaker-and-a-half - highest reliability for critical transmission
Safety Distances
Working space: minimum 1m for 11 kV, 1.5m for 22-33 kV, 2m for 66 kV, 3m for 132 kV. Fence clearances to live parts as per voltage level. Security fencing height ‚â•2.4m with barbed wire.
Fire Protection
Transformer oil containment (110% of largest unit), fire walls between transformers, fixed fire fighting systems for indoor substations, access for fire brigade vehicles.
üéØ Equipment Requirements
Circuit Breakers
Type selection based on fault level, voltage, current rating. Technologies: SF6 (most common), vacuum (MV), air blast (legacy). Operating mechanisms: spring, hydraulic, pneumatic. Rated per IEC 62271 series.
Isolators
Provide visible isolation for maintenance. Must not break load current. Pantograph, centre-break, or vertical-break designs. Interlocking prevents operation under load.
Transformers
Power transformers per AS 60076 series. Cooling types: ONAN, ONAF, OFAF, ODAF. Temperature monitoring, Buchholz relay, pressure relief, oil preservation systems. Typical ratings: 5-500 MVA for distribution.
Current & Voltage Transformers
Instrument transformers per AS 60044. CT accuracy classes: 5P/10P for protection, 0.2/0.5 for metering. VT/CVT for voltage measurement and protection. Burden calculations critical for accuracy.
‚ö° Protection & Control
Protection Philosophy
  • Main and backup protection required for all primary equipment
  • Zone protection principles - overlapping zones
  • Selectivity, sensitivity, speed, and stability requirements
  • Distance protection for transmission lines
  • Differential protection for transformers, busbars, generators
  • Overcurrent and earth fault protection for distribution
Control Systems
SCADA integration, RTU/IED requirements, communications protocols (IEC 61850, DNP3, IEC 104). Hard-wired interlocking for critical functions. Battery systems: 110V DC or 48V DC, autonomy 2-8 hours.
Metering
Revenue metering per AS 62052/62053. Real-time energy management. Power quality monitoring. Load recording and analysis.
üîí Safety Systems
Access Control
Authorized personnel only. Lockout/tagout procedures. Permit-to-work systems for maintenance. Safety signage and hazard warnings.
Personal Protective Equipment
Arc flash PPE requirements per AS/NZS IEC 61482. Arc flash hazard analysis mandatory. Incident energy levels calculated, PPE rated accordingly.
AS 2067:2016 Key Updates
Enhanced arc flash requirements, updated earthing calculations, new sections on SF6 gas management, cybersecurity considerations for digital substations, integration of IEC 61850 communication standards.
Current edition: AS 2067:2016; Referenced standards regularly updated

AEMO Technical Standards
Market Operator
‚ñº
Australian Energy Market Operator - National Electricity Market Requirements
üìã Overview
Authority
AEMO operates the National Electricity Market (NEM) covering Queensland, New South Wales, ACT, Victoria, South Australia, and Tasmania. Also operates Western Australia's Wholesale Electricity Market (WEM).
Regulatory Framework
Technical standards derive from National Electricity Rules (NER), National Electricity Law, and AEMO operating procedures. Mandatory for all registered participants.
üîß Generator Performance Standards
S5.2.5 - Automatic Access Standards
  • Active power control: Ramp rate ‚â•3% per minute of maximum capacity
  • Frequency response: Primary frequency response 0.5-6 seconds, droop 2-7%
  • Reactive power: Power factor 0.9 lagging to 0.95 leading at connection point
  • Voltage control: Maintain ¬±5% of nominal at generator terminals
  • Fault ride-through: Remain connected during voltage dips per specified profiles
S5.2.5.1 - Synchronous Generators
Governor droop settings, AVR response times, short circuit ratio requirements, inertia contribution (H constant typically 3-6 seconds).
S5.2.5.2 - Asynchronous Generators (Wind/Solar)
Fast frequency response (FFR) capability, synthetic inertia requirements, LVRT/HVRT curves, active power recovery post-fault within 1 second to 90% of pre-fault.
üí° Grid Connection Requirements
Connection Process
  • Connection enquiry and application to AEMO/TNSP
  • Preliminary and detailed connection studies
  • Performance standards negotiation (automatic vs. negotiated)
  • Connection agreement execution
  • Commissioning and compliance testing
  • Registration as Market/Network participant
System Strength Requirements
Fault level contribution assessment, minimum three-phase short circuit ratio requirements at connection point. IBR (Inverter-Based Resources) must not degrade system strength below limits.
Power Quality Standards
  • Harmonic limits per AS 61000.3.6 (planning levels)
  • Voltage flicker: Pst ‚â§ 1.0, Plt ‚â§ 0.8
  • Voltage unbalance: negative sequence ‚â§ 1%
  • Interharmonics and supraharmonics assessment for IBR
üéØ SCADA & Communications
Data Provision Requirements
  • Real-time telemetry: MW, MVAr, voltage, frequency (4-second cycle)
  • Status indications: breaker positions, alarms, plant availability
  • Analog measurements: accuracy ¬±1% for revenue-quality data
  • Forecasting data: wind/solar generation forecasts updated hourly
  • Historical data: 13-month retention for market settlements
Communication Protocols
Primary: IEC 60870-5-104, DNP3, or IEC 61850. Dual redundant communications paths required. Encryption and cybersecurity per AEMO's Cyber Security Framework aligned with NIST and IEC 62351.
Control Room Integration
Remote monitoring and control capability to AEMO control room. Automatic Generation Control (AGC) compliance for scheduled generators. 24/7 operator contact requirements.
‚ö° Operational Requirements
Frequency Control
  • Normal operating band: 49.85 - 50.15 Hz
  • FCAS (Frequency Control Ancillary Services) markets: 6-second, 60-second, 5-minute
  • Contingency FCAS requirements based on largest credible contingency
  • Mandatory Primary Frequency Response (PFR) from April 2023
Network Support and Control Ancillary Services (NSCAS)
Voltage control, network loading control, system restart (black start) capability for designated units. Inertia network services in low-inertia regions (SA).
Dispatch and Bidding
Five-minute dispatch interval, 30-minute trading interval. Bid submission 12:30 PM day-ahead. Dispatch conformance: follow AGC signals within ¬±10 MW or ¬±3%.
üåê Renewable Integration Standards
Wind Farm Requirements
Wind forecasting system (AWEFS - Australian Wind Energy Forecasting System). Curtailment capability for system security. Active power control from AEMO. Wake effect modeling in connection studies.
Solar Farm Requirements
Solar forecasting (ASEFS - Australian Solar Energy Forecasting System). Volt-Watt and Volt-VAr control modes. Anti-islanding protection. DC-coupled vs. AC-coupled battery storage considerations.
Battery Energy Storage Systems (BESS)
  • Bi-directional power capability (charge/discharge)
  • Fast frequency response (FFR): 2-second activation to full output
  • State of charge (SoC) management and telemetry
  • Grid-forming capability for system strength contribution
  • Registration as scheduled/semi-scheduled or non-scheduled unit
üìä Metering & Settlements
Metering Installation Requirements
Type 1-4 metering per NER Chapter 7. Type 1 (>100 MW): revenue-grade CT/VT, dual element meters, 5-minute data. Type 4 (<40 MWh/year): basic accumulation meter. SCADA cross-check within ¬±5%.
Market Settlement Data
Metering data provision to AEMO within required timeframes. Settlements run: Preliminary (4 days), Final (28 days), Revision (13 months). Dispute resolution processes for settlement discrepancies.
2025-2026 Focus Areas
Engineering Framework for inertia and system strength, Enhanced Frequency Control Framework, Distributed Energy Resources (DER) integration standards, Offshore wind connection requirements, Hydrogen electrolyzer grid connection standards.
Standards continuously updated via NER rule changes; check AEMO website for latest versions

DNP3 Protocol
IEEE 1815
‚ñº
Distributed Network Protocol - SCADA and automation communication standard
üìã Overview
Official Designation
IEEE Standard 1815-2012 (formerly IEEE 1379). Open protocol developed for reliable communication between SCADA systems, RTUs, IEDs in electric utilities, water/wastewater, and other critical infrastructure.
Global Adoption
Predominant protocol in North America, widely used in Australia, Asia-Pacific. Chosen for robustness over serial and IP networks, efficient bandwidth usage, and extensive event handling capabilities.
üîß Protocol Architecture
Three-Layer Model
  • Application Layer: Request/response messages, unsolicited reporting, event data, file transfers
  • Pseudo-Transport Layer: Message fragmentation/reassembly, sequencing, error detection (CRC)
  • Data Link Layer: Frame structure, addressing (source/destination), master-slave or peer-to-peer modes
Physical Layer Support
RS-232, RS-485, fiber optic, Ethernet, TCP/IP (port 20000), UDP, serial over IP. Operates over diverse media from dial-up to modern IP networks.
üí° Data Object Library
Binary Objects
  • Binary Input (BI) - status monitoring, change events
  • Binary Output (BO) - control points, CROB (Control Relay Output Block)
  • Binary Output Status (BOS) - feedback confirmation
  • Double-bit Binary Input - 4-state values (on/off/indeterminate/invalid)
Analog Objects
  • Analog Input (AI) - measurements: 16/32-bit integer or floating-point
  • Analog Output (AO) - setpoint control
  • Analog Output Status - current setpoint feedback
  • Deadband settings for efficient change reporting
Counter Objects
Accumulated totals (energy, pulses), 16/32-bit with freeze capability. Delta counter for change reporting, binary counter for absolute values.
Time Objects
Time and date stamping with 1ms resolution, time synchronization messages, delay measurement for latency compensation.
File Transfer Objects
Configuration files, firmware updates, event logs, disturbance records. Supports file read, write, delete, directory operations.
üéØ Key Features
Event-Based Reporting
Unsolicited response mode - devices report changes immediately without polling. Reduces network traffic, improves response time. Class 0 (static data), Class 1 (high priority events), Class 2 (normal events), Class 3 (low priority).
Time Synchronization
Master synchronizes all outstations with absolute time. 1ms timestamp resolution, LAN/Delay measurement for propagation compensation. Critical for sequence of events analysis.
Quality Flags
  • ONLINE/RESTART - device status indicators
  • COMM_LOST - communication failure detection
  • OVER_RANGE - measurement exceeded sensor limits
  • REFERENCE_ERR - calibration or reference error
  • LOCAL_FORCED - manually overridden value
Select-Before-Operate
Two-step control mechanism for critical operations. SELECT command arms the output, OPERATE executes. Prevents accidental operations, provides confirmation step. Timeout prevents stale selections.
‚ö° Advanced Capabilities
Data Set Definitions
Prototype (template) and descriptor objects allow custom data groupings. Enables flexible reporting of related parameters. Dynamic dataset modification without firmware changes.
Secure Authentication (SA)
DNP3 Secure Authentication (IEEE 1815.1-2015): Challenge-response authentication, HMAC-based message integrity, session key management, prevents replay attacks, man-in-the-middle, and spoofing. AES-256 encryption optional.
Device Attributes
Self-description capability introduced in DNP3 SAv5. Devices report their configuration, supported objects, capabilities. Facilitates automatic discovery and configuration management.
üåê Implementation Levels
Subset Levels (1-4)
  • Level 1: Basic master/outstation, binary, analog, counters
  • Level 2: Adds controls, time sync, frozen data
  • Level 3: Adds file transfer, datasets
  • Level 4: Full protocol implementation including all optional features
Device Profiles
XML-based Device Profile Documents specify exact implementation. Published profiles ensure interoperability. Testing labs verify conformance to profiles.
üìä Applications
Electric Utilities
  • Substation automation and RTU communication
  • Distribution SCADA systems
  • Transmission EMS/SCADA integration
  • IED data concentration
  • AMI/Smart Grid backhaul
Water/Wastewater
Pump station control, reservoir level monitoring, treatment plant automation, pipeline pressure management, flow metering.
Oil & Gas
Pipeline monitoring, compressor station control, custody transfer metering, leak detection systems.
Renewable Energy
Wind farm SCADA, solar plant monitoring, battery storage management, hydro plant control.
üîí Cybersecurity Considerations
DNP3 SAv5 Security Features
  • User authentication with role-based access control
  • Message authentication codes (MAC) for integrity
  • Challenge-response prevents replay attacks
  • Aggressive mode for time-critical messages
  • Key update procedures for periodic renewal
Network Security Layers
VPN tunneling, TLS/SSL wrappers, firewall rules for port 20000, IDS/IPS monitoring, network segmentation, compliance with NERC CIP standards.
DNP3 vs IEC 104 vs IEC 61850
DNP3 excels in reliability over poor networks, extensive event handling, North American deployment. IEC 104 common in Europe/Asia, lighter protocol. IEC 61850 offers richer data models, peer-to-peer, ideal for modern digital substations. Many systems support multiple protocols.
Current version: IEEE 1815-2012; Secure Authentication: IEEE 1815.1-2015; Active development by DNP Users Group
